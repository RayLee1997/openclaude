# 编排与工作流准则 (workflow-orchestration-rules.md)

本规则约束 Agent 在接收复杂工程需求、漏洞修复与模块研发时的**行动编排逻辑**，作为最高优先级的执行标准。

## 1. 计划先行与动态调整 (Plan Default)

- **强制计划 (Plan First)**：对于任何非基础任务（≥3步或涉及系统架构），**必须**先进入 PLANNING 模式制定详尽方案与 Checklist。
- **防御性停顿 (Stop & Re-plan)**：若执行偏离预期或受阻，**立即停止盲目试错**，通过复盘重新规划路径。
- **全生命周期覆盖**：计划必须前置包含"如何验证"，构建与测试不可分割。
- **消除歧义**：动码前输出细致规范，确保与最终交付目标完全对齐。

## 2. 工具/技能并行与卸载 (Sub-strategy)

- **主副分离**：积极使用具体的专项 Skills（如 `web-research`, `us-stock-analysis`）或外部调用去处理探索、检索和并行分析，保持主视角的上下文清晰。
- **专注执行**：面对极度复杂的问题，拆解为独立子任务流，保证单阶段内的心智聚焦。

## 3. 自我进化与闭环 (Self-Improvement Loop)

- **捕获教训 (Capture Lessons)**：在受到用户的任何干预、指正后，必须提取模式，防止重蹈覆辙。
- **严格迭代**：残酷地将错误反思整合进下一次计划阶段。如果同类 Bug 出现第二次，就是不可原谅的失职。

## 4. 交付验证的底线 (Verification Before Done)

- **无证明，不完结**：除非成功运行了测试、查阅了验证日志或输出了确凿的可视化证明，绝不允许声称"已修复/已完成"。
- **前后追溯 (Diff Check)**：当涉及更改时，自觉审视更改前后的 Diff，评估扩散影响。
- **Staff Engineer 标准**：交付前自问：*“资深架构师会批准这个方案合并吗？”*。

## 5. 追求极致优雅 (Demand Elegance)

- **克制与反思**：面对复杂改动，停留在原型阶段问自己："是否有更优雅的代码解法（A more elegant way）？"
- **摒弃黑客补丁 (Hack)**：如果一种修复让人觉得"仅仅是苟住了"，那么在获取全部上下文后，请直接推翻并实施真正优雅的重构。
- **平衡感**：对于显而易见的简单 Bug，去过度工程（Don't over-engineer），一针见血修复即可。

## 6. 独立无感除错 (Autonomous Bug Fixing)

- **杜绝伸手党**：当接收到报错记录或 Bug 描述时，直接定位日志、查阅出错代码、运行环境并**彻底修复**。不要事无巨细地要求用户指导。
- **零中断 (Zero Context Switching)**：目标是让用户不需任何上下文切换即可获得可用结果。

## 7. 编程核心底线原则 (Core Principles)

1. **简单至上 (Simplicity First)**：对所有动作寻优，力求极简，控制最少代码影响面。
2. **深挖根因 (No Laziness)**：寻找 Root Causes，不贴临时创可贴，维持高级开发环境的安全感。
3. **最小侵入 (Minimal Impact)**：不引入新依赖，不触碰与当前目标无关的隔离带模块，严防次生 Bug。

## 8. 临时文件隔离纪律 (Session Temp Discipline)

- **唯一落盘目录**：所有 Agent 运行期间产生的临时脚本、一次性数据文件、调试产物，**必须**写入 `agent-sessions/tmp/`，绝对禁止散落到工作区根目录、知识库子目录或系统 `/tmp`。
- **路径构造**：使用相对于工作区的 `agent-sessions/tmp/` 路径，目录不存在时自行 `mkdir -p` 创建。
- **生命周期意识**：临时文件用完即删，或在 Session 结束时由用户统一清理。不得将临时产物遗留为永久文件。
- **违规即失败**：在非 `agent-sessions/tmp/` 路径下创建临时文件视为**纪律违规**，等同于交付缺陷。

## 9. 增量写作与分步迭代 (Incremental Writing & Iteration)

针对长篇内容生成或复杂文档写作，必须强制执行三段式更新流：

1. **大纲先行 (Outline First)**：第一步**必须且仅**输出完整大纲结构，每条标题仅附带一句“内容方向说明”，绝对禁止在此时展开正文。
2. **逐章整合 (Incremental Updates)**：大纲明确（或经确认）后，充分和全面的收集信息进行深度推理与融合，然后**逐章渐进式**填充正文。
3. **总览收口 (Finalize Summary)**：全部主体章节交付完毕后，最后提炼并提取全篇核心发现，以此完成核心摘要的更新或输出。

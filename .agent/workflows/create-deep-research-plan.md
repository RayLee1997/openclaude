# 深度调研计划创建工作流

将用户的调查主题转化为 **模块化、可并行执行** 的调研计划，产出机构级深度调查报告。

> **适用场景**: 地缘政治、经济调研、行业调查、公司尽调、技术趋势等多维度深度调查。

## Phase I: 调研计划设计 (Planning)

### 0. 需求确认

询问用户（已提供则跳过）：调查主题、时间范围、核心焦点、输出位置（默认 `07_Investigation/{主题名}/`）、深度偏好（默认机构级）。

### 1. MECE 主题拆解

// turbo

对调查主题进行 MECE 拆解，识别 8-14 个独立模块。参考维度框架：

| 调查类型 | 推荐维度 |
| --- | --- |
| **经济/国家** | 宏观经济、财政、货币、经济结构、社会民生、战略行业、外债/信用、政治经济学、国际关系 |
| **地缘政治** | 历史溯源、军事态势、核心冲突、代理人网络、大国博弈、能源安全、国际法、国内危机 |
| **行业调查** | 产业链全景、竞争格局、技术路线、监管环境、市场规模、关键玩家、投资趋势 |
| **公司尽调** | 商业模式、财务健康、竞争优势、管理层、风险因素、估值、行业地位 |

每个模块定义：① 模块名称（中+英）② 核心问题（2-3 个）③ 优先级（P0/P1/P2）

### 2. 编号文件结构

```text
{调查目录}/
├── {主题名}_调研计划.md             ← 📌 本工作流产出
├── 00_阅读索引.md                   ← 🔒 最后更新
├── 01_核心摘要.md                   ← 🔒 全部模块完成后
├── 02-{N-2}_{调研模块}.md           ← ✅ 逐章撰写
├── {N-1}_融合分析.md                ← 🔒 全部模块完成后
├── {N}_终极研判.md                  ← 🔒 融合分析完成后
├── {N+1}_附录A_{对比矩阵}.md        ← 📎 跟随对应模块
├── {N+2}_附录B_参考文献总库.md       ← 📎 汇总全局文献
├── {N+3}_附录C_术语表.md            ← 📎 持续积累
└── {N+4}_附录D_核心数据溯源库.md     ← 🔴 记录每章核心数据的具体来源与 API 调用参数，确保结论 100% 可复现
```

> [!IMPORTANT] 文件名锚定规则
> 此结构中的**每一个文件名**（包括编号、下划线、中文名、英文名）都是后续执行的**唯一权威来源**。Phase II/III 创建文件时，**必须**从调研计划的文件结构章节中逐字复制文件名，**严禁**凭记忆或缩写输入。

### 3. 设计并行工作流 (Workstream)

将模块按逻辑相关性重组为 **3-5 条 Workstream**。设计原则：最小依赖 / 角色明确 / 数据源隔离 / 标准化产出。

#### 数据源选型路由 (Data Routing)

1. **宏观经济 (FRED-IMF-WB 三角)**：美国高频→`fred-data` / 全球货币→`imf-data` / 全球发展→`worldbank-data` / 中国→`cn-stats-data`
2. **个股投研 (三角验证)**：美股→`us-stock-analysis`(EDGAR+YF)+`fred-data`+Brave / A股→`cn-stock-analysis`(akshare)+`cn-stats-data`+Brave / 估值→`create-valuation-model-plan`
3. **地缘调查 (定量先行)**：经济体量→`worldbank-data`+`owid-data` / 贸易债务→`imf-data`(BOP/DOT/GDD) / 亚太→`adb-data`
4. **定性补充**：所有数据均配合 `web-research` (Brave Search) 承担最后一公里信息补全。

#### 每条 Workstream 必须包含

角色 Persona / 覆盖模块 / 调研范围表（子课题×问题×优先级）/ 核心数据源清单 / 标准化产出清单 / 向融合层传递的评级数据。

必须生成 Mermaid flowchart 展示 WS 依赖关系（实线→融合层，虚线-.- >弱依赖），使用 Healing Dream 配色。

### 4. 融合层定义

由"首席研究员"执行。包含：核心摘要(01) / 融合分析 / 终极研判(Bull/Base/Bear) / 附录 / 阅读索引(00)。

### 5. 标准化报告模板

```markdown
---
date: {当前日期}
status: Complete
tags: [{标签}]
parent: "[[{调研计划}]]"
---

# [NN]_[模块标题]

## 核心发现 (Key Findings)
> [!NOTE] 本章核心
> [≤200 字核心结论]

## 详细分析
### [子章节]
[正文 + 数据表 + Mermaid]

## 评级输出
| 维度 | 评级 | 置信度 | 依据 |
| --- | --- | --- | --- |
| [维度] | 🔴/🟠/🟢 | 高/中/低 | [理由] |

## 📚 关联文献与参考资料
> [!INFO] 数据溯源
>
> **{分类标题A}**：
>
> - {来源机构} ({年份})：《{标题}》（{关键数据摘要}），[原文链接]({url})
>
> **{分类标题B}**：
>
> - {来源机构}：{简要说明}
>
> **前序模块交叉引用**：
>
> - [[{模块名}]]: {引用要点说明}

[^1]: 调用参数快照：`{"query": "..."}`，抓取时间：YYYY-MM-DD。【数据溯源：见附录 D.x】
```

> [!CAUTION] 参考文献格式铁律
>
> 1. **所有内容在 `> [!INFO]` callout 内**：bullet list、bold 标题、交叉引用全部用 `>` 前缀
> 2. **分类用 `**粗体**：`**，**禁用 `###` H3**（callout 内 H3 不渲染为标题）
> 3. **`[^N]:` 脚注在 callout 外**：Obsidian 脚注必须在顶层才能正确渲染悬浮预览
> 4. **禁用表格**：callout 内表格宽度受限且无法横滚，改用 bullet list + 括号注释
> 5. **统一 emoji**：标题必须是 `## 📚 关联文献与参考资料`，不可省略 📚

### 6. 引用与质量规范

**引用规范**：① 可见内联链接 `[Title](URL)`（⚠️严禁 reference-link 语法）② 脚注 `[^1]` ③ MCP 伪协议溯源 `mcp://...`

**数据源分级**：Tier 1 官方机构 → Tier 2 智库(IMF/WB) → Tier 3 权威媒体(Reuters/FT) → Tier 4 本地媒体 → Tier 5 OSINT

**质量检查表**（章节自审 + 全局终审统一使用）：

```markdown
- [ ] 首尾架构守卫：YAML Frontmatter + 参考资料脚注块完整
- [ ] 数据一致性：各模块同一指标数据一致
- [ ] 评级校准：🔴/🟠/🟢 标准对齐
- [ ] 数据溯源：各模块末尾含完整引用，汇总至附录B
- [ ] 数据溯源库：每章核心数据的指标名、Skill/API 调用参数、数据源链接、抓取时间已追加至附录D
- [ ] 双语规范：技术术语英文、综合叙述中文
- [ ] Mermaid 配色：统一 Healing Dream
- [ ] Mermaid Key Insights：每个图表下方含 2-4 条数据驱动的关键洞察
- [ ] 时间锚定：所有"当前"描述锚定调研日期
- [ ] 核心摘要覆盖：01 覆盖各模块核心发现
```

### 7. 关键问题清单

为融合层定义 **10-15 个跨模块关键问题**，必须跨越多模块、涉及结构性判断、含历史对标/国际比较。

### 8. 输出调研计划 → 用户审批

整合为调研执行大纲 `{主题名}_调研计划.md`，包含：设计原则 / 文件结构 / Mermaid 依赖图 / WS 详细拆解 / 融合层 / 角色-技能-工具匹配表 / 模板 / 质量检查表 / 关键问题 / 执行建议。

提交 Review；用户确认 → Phase II；用户反馈 → 迭代优化。

### 9. 🆕 生成 Execution Manifest (执行清单)

> [!CAUTION] 关键步骤 — 防止计划与执行脱钩
> 此步骤是 Phase I → Phase II 的**唯一桥梁**。跳过此步骤将导致执行偏差。

用户审批通过后、Phase II 开始前，**必须**执行以下步骤：

1. **打开调研计划文件**，定位"文件结构"章节（`## 三、文件结构` 或等价标题）
2. **逐行提取**所有文件名（包括编号前缀、下划线、完整中文名），生成 **Execution Manifest**
3. **写入 task.md**，格式为逐文件的 checkbox 清单：

```markdown
# {主题名} Execution Manifest

## Source of Truth
- 调研计划文件: `{调研计划路径}`
- 文件结构章节: 第 {N}-{M} 行

## Phase II: 调研模块 (逐文件产出)
- [ ] `02_{模块1精确文件名}.md`
- [ ] `03_{模块2精确文件名}.md`
- [ ] ...
- [ ] `{N-2}_{最后模块精确文件名}.md`

## Phase II: 附录
- [ ] `{N+1}_附录A_{精确名}.md`
- [ ] `{N+2}_附录B_参考文献总库.md`
- [ ] `{N+3}_附录C_术语表.md`
- [ ] `{N+4}_附录D_核心数据溯源库.md` ← 🔴 逐章追加，记录核心数据出处与查询参数

## Phase III: 融合层 (全部模块+附录 Complete 后)
- [ ] `{N-1}_融合分析.md`
- [ ] `{N}_终极研判.md`
- [ ] `01_核心摘要.md`
- [ ] `00_阅读索引.md` ← 最后更新
```

1. 在 task.md 中嵌入 **Anti-Drift Anchor**：

```markdown
> [!CAUTION] Anti-Drift Anchor
> 创建每个文件时必须从上方清单中 **复制粘贴** 精确文件名。
> 禁止：缩写、省略前缀、更改分隔符、合并多个模块为一个文件。
> 每完成一个文件立即标记 `[x]`。
> 每完成一个 Workstream 后执行 Phase Gate 校验。
```

## Phase II: 调研执行 (Execution)

> [!IMPORTANT] 核心原则
> **逐章撰写 → 可视化 → 校验 → 完善 → 下一章。** 01/融合/终极研判 禁止在所有模块完成前撰写。00 是最后更新的文件。**所有文件名必须从 Execution Manifest 复制粘贴**。

### 10. 逐章执行循环

按优先级 P0→P1→P2 对每个模块 (`02`-`{N-2}`) 执行 5 步循环：

**Step A: 骨架先行** — 严禁一次流式生成全文！先用工具创建空壳文件（顶部 YAML + 底部参考资料占位符）。**⚠️ 文件名必须从 task.md Manifest 复制。**

**Step B: 调研与正文注入** — // turbo

1. 按调研范围表逐项检索。
2. **强制多源交叉验证**：宏观→FRED-IMF-WB 三角 / 美股→SEC-Yahoo-Brave 三角(SEC 为准) / A股→akshare-NBS-Brave 三角。
3. 使用对应 Agent Skill 获取数据，严格数据溯源。
4. 用 `replace_file_content` 将正文注入骨架中间。
5. **数据溯源登记**：将本章使用的核心数据指标、Skill/API 调用参数、数据源 URL 及抓取时间，追加写入 `{N+4}_附录D_核心数据溯源库.md`。

**Step C: 图表解耦注入** — 正文完成后独立生成 Mermaid 图表（遵守 Healing Dream 配色）：

| 数据类型 (关联数据源) | Mermaid 类型 |
| --- | --- |
| 时序趋势 (FRED, NBS, WB) | `xychart-beta` |
| 结构占比 (营收, OWID 能源) | `pie` |
| 流程/因果链 | `flowchart` |
| 概念层级/拆解框架 | `mindmap` |
| 资金/贸易流向 (IMF BOP) | `sankey-beta` |
| 历史演进/地缘事件 | `timeline` |

每模块至少 1-3 个图表，用编辑工具精准注入。

> [!IMPORTANT] 图表关键洞察 (Key Insights) — 强制要求
> **每个 Mermaid 图表下方必须紧跟 Key Insights 分析块**，遵循 `mermaid-chart` 技能第 4 步规范。格式如下：
>
> ```markdown
> **Key Insights:**
> - 🏆 **[关键点]**: [数据] + [解释]
> - 📈 **[趋势]**: [数据] + [解释]
> - ⚠️ **[风险/关注]**: [数据] + [解释]
> ```
>
> **规则**：
>
> - 每个 Key Insights 块 **2-4 条**，禁止空泛描述，必须引用图表中的具体数据
> - 使用 emoji 前缀区分类型：🏆 核心发现 / 📈 趋势 / ⚠️ 风险 / 💡 洞察
> - Key Insights **不是图表标题的重复**，而是从图表数据中提炼的**非显而易见的推论**
> - 缺少 Key Insights 的图表视为**不完整**，在质量检查中标记为未通过

**Step D: 格式校验** — 执行质量检查表（见 Step 6）。🚨首尾截断立即修复。

**Step E: 补充完善** — 通过→标记 Complete（在 task.md Manifest 中标记 `[x]`）；未通过→回 B/C 补充（最多 2 轮，超限用 `[!CAUTION]` 标注留融合阶段处理）。

### 11. 🆕 Workstream Phase Gate (每个 WS 完成后)

每完成一个 Workstream，**必须**执行以下校验：

```
1. list_dir → 获取调研目录实际文件列表
2. 对比 task.md Manifest 中当前 WS 的文件名
3. 检查项:
   - [ ] 文件数量一致
   - [ ] 每个文件名精确匹配（包括编号、下划线、中文）
   - [ ] 无意外的合并文件（一个计划模块 = 一个独立文件）
   - [ ] 无意外的缺失文件
4. 发现偏差 → 立即修正 → 然后才能进入下一个 WS
```

### 12. 附录撰写

附录 A (对比矩阵) / 附录 B (文献总库，汇集各模块引用) / 附录 C (术语表) / **附录 D (核心数据溯源库)**。

> [!IMPORTANT] 附录 D — 核心数据溯源库
> 此文件**不是最后统一撰写**，而是在 Phase II 执行每一章时**逐步追加**。每完成一章的 Step B（调研与正文注入），须将本章使用的所有核心数据条目追加至附录 D，格式如下：
>
> | 章节 | 指标名称 | 数据值 | 数据源 | Skill/API 调用 | 调用参数快照 | 抓取日期 |
> | --- | --- | --- | --- | --- | --- | --- |
> | 02 | TFR (中国) | 0.97 | UN WPP 2024 | `worldbank-data` | `SP.DYN.TFRT.IN, CHN, 1995-2025` | 2026-03-02 |

**⚠️ 附录文件名使用连续编号 ({N+1}, {N+2}, {N+3}, {N+4})，不使用 A1/A2/A3 之类的替代编号。从 Manifest 复制粘贴。**

## Phase III: 融合与终稿 (Synthesis)

### 13. 🆕 Phase III 进入条件检查 (Gate)

> [!CAUTION] 强制校验 — Phase III 开始前必须执行
> 此步骤不可跳过。如果检查不通过，必须返回 Phase II 补完缺失文件。

```
1. 打开调研计划原文，定位文件结构章节
2. list_dir → 获取实际文件列表
3. 逐项对比:
   - [ ] 所有 02-{N-2} 模块文件存在且命名正确
   - [ ] 附录 A/B/C 文件存在且编号正确
   - [ ] 无计划外的合并/拆分/重命名
   - [ ] task.md Manifest 中所有 Phase II 项已标记 [x]
4. 不通过 → 返回 Phase II 修正 → 重新执行此检查
5. 通过 → 开始 Phase III
```

### 14. 融合分析 → 终极研判 → 核心摘要 → 阅读索引

1. **融合分析 ({N-1})**: 通读各模块核心发现→识别跨模块耦合→历史对标→回答关键问题→Mermaid 可视化→自审
2. **终极研判 ({N})**: 情景推演(Bull/Base/Bear)→最终判断+置信度→自审
3. **核心摘要 (01)**: 提炼 6-10 条 Key Findings→结论矩阵→一句话 Verdict→推荐阅读路径→自审
4. **阅读索引 (00)**: 🔒最后更新。报告属性表→阅读路径(⚡15min/📖60min/🔬3hrs)→章节导航→Mermaid 结构图→数据源层级→局限性声明→最终全局自审

**⚠️ 每个文件名从 Manifest 复制粘贴。完成一个标记 `[x]`。**

## Phase IV: 质量终审

### 15. 最终交付校验

```
1. list_dir → 获取完整文件列表
2. 对比 Manifest → 确认全部文件产出且命名正确
3. 执行全局质量检查表（Step 6）
4. 确认所有文件完整
5. 如涉及新调查主题，酌情更新 GEMINI.md
```

> [!CAUTION] 关键时序约束
>
> - 01 + 融合 + 终极研判 **禁止**在所有调研模块完成前撰写
> - 00 **必须**是最后更新的文件
> - 每个模块完成后**立即**执行可视化，不要积压
> - **所有文件名必须从 Execution Manifest 复制粘贴，严禁凭记忆输入**
> - **一个计划模块 = 一个独立文件，严禁合并**
